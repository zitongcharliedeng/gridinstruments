# Coding Standards

These rules enforce the project's heavenly restrictions. Each rule is an ast-grep YAML pattern that catches structural antipatterns in the tangled TypeScript. They run as part of the literate-state-machine-wiki lint stage.

Rules that protect SPECIFIC components live in those component's `.lit.mdx` files. These are the UNIVERSAL rules that apply everywhere.

## No Absolute In Scrollable

position:absolute inside overflow:auto container — the element will render at BOTH positions in some browsers. Use position:sticky instead.

```{.yaml file=ast-grep-rules/no-absolute-in-scrollable.yml}
id: no-absolute-in-scrollable
language: css
rule:
  all:
    - pattern: "position: absolute"
    - inside:
        any:
          - has:
              pattern: "overflow-y: auto"
          - has:
              pattern: "overflow: auto"
        stopBy: end
files:
  - "**"
  - "index.html"
severity: error
message: "position:absolute inside overflow:auto container — the element will render at BOTH positions in some browsers. Use position:sticky instead."
note: "See #272: Chrome renders absolute-positioned children at scroll position AND absolute position when parent has overflow-y:auto."
```
